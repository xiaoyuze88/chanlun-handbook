# 附录C 操作规则审核方案

> **文档用途**：独立执行文档，完整包含所有必要上下文，可直接喂给 AI 在新对话中执行。
> **目标文件**：`organized-v2/附录/C-操作规则总表.md`
> **输出文件**：`v2-systematic/QA-C-REPORT.md`

---

## 一、背景与目标

**organized-v2** 是缠中说禅缠论知识库的最终产物，包含30章正文 + 总论 + 附录A~E。
附录C（`organized-v2/附录/C-操作规则总表.md`）收录了全部约75条操作规则，按场景分组，格式为：

```
**规则ID：P{篇章}-{章节}-R{序号}（规则名）**
前提：……
触发：……
操作：……
冲突优先级：……
来源章节：[链接]
```

**审核目标**：不是检查格式，而是评估**内容质量**——规则是否可执行、是否完整、是否有内在矛盾。

---

## 二、审核维度

### Q1：触发条件是否具体可判断？

**问题**：触发条件如果过于模糊（如"出现信号"、"走势变化"），AI 或人工无法据此执行。

**判断标准**：触发条件应能回答"在 K 线图上，看到什么具体现象时，这条规则启动"。

**需标记为 ⚠️ 的情形**：
- 触发条件使用了"疑似"、"倾向于"、"可能"等主观判断词，但没有提供判断依据
- 触发条件引用了另一个规则中定义的概念，但没有说明如何识别
- 触发条件的边界模糊（如"距买点形成位置不远"，"不远"无量化依据）

---

### Q2：操作步骤是否无歧义？

**问题**：操作步骤应该是可机械执行的，不依赖额外判断。

**判断标准**：每个步骤应明确"做什么"，步骤间逻辑清晰，没有隐含的跳步。

**需标记为 ⚠️ 的情形**：
- 操作中含有"根据情况"、"视走势"、"需判断"等表述，但没有分支枚举
- 步骤顺序存在歧义（做A之前是否必须做B？）
- 步骤的终止条件不明确（持有到什么时候？等待多久算失效？）

---

### Q3：冲突优先级是否完整覆盖实际冲突？

**问题**：多个规则可能同时适用于同一市场状态，冲突优先级应明确说明哪条规则胜出。

**判断标准**：附录 C 末尾已有"待确认冲突"区域（列了3条），但正文中的冲突声明可能有遗漏。

**需标记为 ⚠️ 的情形**：
- 某规则声称"优先于X"，但 X 并不存在于附录 C 中
- 两条规则在相同前提下指向不同操作，但二者都没有声明优先级
- 冲突声明使用了"本规则让位于……"，但没有说明"让位"后具体执行哪条

**重点检查对**（已知存在或疑似存在冲突）：
- P3-10-R2（差价操作）vs P3-10-R3（满仓持有）
- P4-13-R1（操作级别参与范围）vs 任何小级别买卖点规则
- P5-19-R1（同级别顶/底背驰绝对退出）vs 持仓中的所有其他规则
- P7-27-R1（禁止杠杆，最高优先级）与其余规则的关系

---

### Q4：前提条件是否可以被核实？

**问题**：前提条件是规则的"守门员"，如果无法判断前提是否成立，规则就无法触发。

**判断标准**：前提条件中引用的概念（如"走势中枢已形成"、"趋势已确认"）应在附录A（定义）中有明确定义，或规则本身已给出判断方法。

**需标记为 ⚠️ 的情形**：
- 前提中的核心概念在附录A中找不到对应定义
- 前提中使用了多个条件的 AND/OR，但没有说明优先判断顺序

---

### Q5：规则覆盖是否有明显空白？

**问题**：查看是否存在重要的操作场景，但没有对应规则。

**已知的场景分组**（附录C自述）：
- 建仓（找买点入场）
- 离场（卖点退出）
- 风控（止损/杠杆）
- 级别（操作级别选择与切换）
- 仓位（差价/零成本/资金管理）

**需标记为 ⚠️ 的情形**：
- 某个重要场景（如"换股"、"空仓等待"、"强制性止损边界"）没有任何规则覆盖
- 某篇章（P1~P8）的规则数量明显偏少，与该篇章的内容复杂度不匹配

## 三、执行指令

### 输入文件

完整读入以下文件：
1. `organized-v2/附录/C-操作规则总表.md`（主体，约750行，75条规则）
2. `organized-v2/附录/A-定义速查.md`（参考，用于 Q4 前提核实）

不需要读入其他文件。

### 执行方式

**逐条审核**：对 C-操作规则总表.md 中每一条规则，逐一检查 Q1~Q4。

**每条规则输出格式**：
```
**P{X}-{YY}-R{Z}**：{规则名}
- Q1 触发：✅ 可判断 / ⚠️ {问题描述}
- Q2 操作：✅ 无歧义 / ⚠️ {问题描述}
- Q3 冲突：✅ 覆盖完整 / ⚠️ {问题描述} / N/A（无冲突声明，可接受）
- Q4 前提：✅ 可核实 / ⚠️ {问题描述}
```

**Q5 在逐条审核完成后单独输出**：列出发现的场景空白。

将完整报告写入：`v2-systematic/QA-C-REPORT.md`

## 四、判断标准参考

为帮助审核时统一判断尺度，以下是"可接受"vs"需标记"的边界举例：

| 表述类型 | 示例 | 判断 |
|---------|------|------|
| 引用已定义概念 | "确认趋势背驰成立（见 P5-16-R3）" | ✅ 可接受，已有前置规则 |
| 步骤内含分支且枚举了全部分支 | "若X则A；若非X则B" | ✅ 可接受 |
| 模糊的量化条件 | "距离不远可介入" | ⚠️ 需标记 |
| 主观判断但有依据 | "用MACD面积目测估算" | ✅ 可接受，已说明方法 |
| 冲突存在但有声明 | "本规则优先于P3-10-R2" | ✅ 可接受 |
| 冲突存在但无声明 | 两条规则在相同场景下无优先级说明 | ⚠️ 需标记 |
| 前提概念在附录A中有定义 | "趋势已确认（≥2个同向不重叠中枢）" | ✅ 可接受，已自含判断标准 |
| 前提概念无处查找且无解释 | "走势结构已被操作级别缠论框架解读" | ⚠️ 需标记 |

## 五、注意事项

1. **不要修改附录C本身**，只输出报告。
2. **不要评价缠论理论本身的正确性**，只评估规则的可执行性和完整性。
3. **保守标记**：如果一个表述在专业上下文中有约定俗成的理解（如缠论中的"第一类买点"），不因其未被文字完整定义而标记为 ⚠️。
4. 附录C末尾"待确认冲突"区域是已知问题列表，审核报告中可直接引用，但应说明是否有更多未列出的冲突。
5. **报告语言**：中文。
