# concept-review

## Overview

多模型并行复核缠论知识库概念，每个模型独立执行完整复核流程（读原文→定位可疑点→溯源原文→汇报），主会话横向对比后汇总。

## 三种模式

| 模式 | 模型数 | 模型列表 |
|------|--------|---------|
| 标准 | 3 | kimi-k2.5-ioa, glm-5.1-ioa, minimax-m2.7-ioa |
| 深度 | 5 | 标准3 + gpt-5.4, gemini-3.1-pro |
| 超深度 | 6 | 深度5 + claude-sonnet-4.6-1m |

## 执行步骤

### 1. 确认模式

AskUserQuestion 让用户选 3/5/6（用户已指定则跳过）。同时准备任务包：用 graphify query 定位相关 organized-v2 文件。

### 2. 并行发起 subagent

单条消息并行所有 Agent 调用，`run_in_background: true`，`model` 参数指定模型。每个 subagent prompt：

```
> Model: {模型ID}

你是一位缠论概念复核专家，负责对知识库中的概念表述进行严格复核。
规则：每个问题必须有原文依据，不凭感觉判断；只读不写，不修改任何文件。

## 复核任务

步骤1：读取 organized-v2 原文
- 读取：{文件列表}
- 先检查 <!-- 前置知识 --> 注释，有则先读前置知识文件
- 识别该概念的定义、所有相邻概念、已有注释

步骤2：定位可疑点
- 找同一文件内两个或多个相邻概念，判断它们之间的关系是否说清楚了
- 重点看：充要条件 vs 必要条件；触发条件 vs 内部结构；定义 vs 操作规则

步骤3：溯源原文（必做）
- 查 organized-v2 文件末尾的"出处索引"表格，找到对应课文编号
- 读 archived/108/ 对应课文，与 organized-v2 表述逐一对比
  - 文件命名：{4位序号}-{hash}-{3位课号}.md
  - 用 ls /Users/xyz/Sites/private/chzhshch-108-plus/archived/108/ | grep "{序号}" 定位
- 若仍有不确定，可查 archived/graphify-out-108/graph.json 或 archived/organized-v1/

步骤4：汇报结论
用 SendMessage 发给 main，格式如下：

Model: {模型ID}

## 总体评估
[整体准确性，主要关注点，做得好的地方]

## 问题清单

### 🔴 必须修复
- **位置**：文件名 § 章节名（或行号）
- **问题**：[具体描述，引用 organized-v2 原文 vs archived/108 原文对比]
- **建议**：[改成什么样，原文依据]

### 🟡 建议修复
（同上格式）

### 💭 小改进
（同上格式）

## 无问题确认
[若某部分核实无误，明确说明]

## 溯源记录
[列出读取的 archived/108 文件路径及关键原文段落]
```

### 3. 等待全部结论

SendMessage 回传。5 分钟查一次 TaskOutput：running 继续等，failed/cancelled 标注缺失。

### 4. 汇总评估

横向对比结论，按以下规则处理：

- **多 agent 共识问题（≥2 份）**：直接纳入修改清单
- **单 agent 独家问题**：主会话必须独立复核（读原文验证），不可直接采信
- **🔴 必须修复**：无论共识度如何，主会话均独立读原文确认后再执行

### 5. 向用户汇报

综合判断是否需要改、改哪里、怎么改，等用户确认后写入文件。

### 6. 更新记录

复核完成后同步更新以下两个文件：

**`docs/concept-review/PROGRESS.md`**：在复核列表追加一行
```
| {序号} | {概念名} | {对应文件} | {状态} | {结论摘要} | {日期} |
```
- 状态：✅ 已复核，无问题 / 🔧 已复核，已修改
- 结论摘要：如"3项修改（详见修改记录#N）"或"无需修改"

**`docs/concept-review/CHANGELOG.md`**：追加完整修改记录，格式：
```
## #{序号} {概念名}｜{文件}｜{日期}

**复核模型**：{模型ID列表}

### 已执行修改

**[字母] severity 文件 章节: 简述**
- 共识度：{哪些agent同意 / X/Y}
- 原有表述：`原文`
- 问题：{描述，附原文依据}
- 修改：{改成了什么}

### 单 agent 独家问题的主会话复核记录

**[字母] {agent} 提出的问题**
- 提出方：{模型ID}（{severity}）
- 主会话复核：{独立验证推理}
- **复核结论：不成立 / 采纳 / 已被其他修改覆盖**
```

## 注意事项

- 所有 subagent 必须并行发起（同一批次 Agent 调用），不要串行
- 如果某个模型出错或超时，跳过该模型继续汇总其他结果，标注缺失
- 汇总时不要简单罗列，要有综合判断和优先级排序
